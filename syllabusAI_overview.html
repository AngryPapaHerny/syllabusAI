작동 방식

3개의 계층으로 동작합니다

학습자가 쓰는 빠른 웹앱, 자산이 쌓이는 데이터 계층, 콘텐츠를 만드는 생성 엔진을 분리해 비용과 속도를 동시에 잡습니다.

A · 학습 런타임

학습자가 만나는 웹앱 (Next.js)

온보딩·진단부터 학습·평가·재학습까지. 가볍고 빠르게, 이미 만들어진 콘텐츠를 조립해 보여줍니다.

핵심 차별점

"AI로 콘텐츠 생성"은 누구나 합니다.
우리는 6가지를 다르게 했습니다

탭을 눌러 각 설계 원칙을 확인하세요.

생성 엔진

틀린 걸 가르치지 않는 4단계 파이프라인

교육에서 가장 위험한 건 "AI가 자신 있게 틀리는 것". 모든 콘텐츠는 검증을 통과해야 학습자에게 도달합니다.

1

난이도 보정 (Calibrator)

주제를 원자 개념으로 분해 → 학습자 선수지식과 비교 → 분량 대비 인지 부하 적정성 추정.

2

환각 방지 (Grounding)

공식 문서·교과서 등 신뢰 소스 검색, 코드 예제는 실제로 실행해 확인, 모든 사실에 출처를 부착.

3

P-C-S-M-A 작성 (Writer)

동기(P)→개념·비유(C)→실습(S)→응용 미션(M)→평가(A) 순서로 하나의 완결된 학습 유닛 작성.

4

자체 검수 (Self-Check)

정답이 테스트를 통과하는가, 퀴즈에 복수 정답은 없는가, 난이도가 목표에 맞는가 — 실패 시 되돌려 재생성.

🔌 LLM Gateway — 특정 AI에 종속되지 않습니다

생성·튜터·채점의 모든 호출이 단일 추상 계층을 거칩니다. 작업별로 최적 모델을 정책으로 선택하고, 장애 시 자동 폴백하며, 새 모델이 나오면 코드 수정 없이 교체합니다.

진짜 자산은 '커리큘럼'이 아니라 재사용 가능한 원자 개념(Concept) 라이브러리입니다. 새 학습자의 경로 대부분이 기존 유닛 조립으로 즉시 충당되고, 빈 칸만 새로 생성합니다.

🧩 개념 단위 재사용

"코딩테스트 준비"와 "파이썬 알고리즘 기초"는 이름은 달라도 다수 개념이 동일. 개념 단위로 쪼개 매칭하면 재사용률이 급등합니다.

🎚 수준별 변형 캐싱

한 개념을 입문/중급, 비유 중심/코드 중심 등 변형으로 저장. 재학습 시 같은 설명을 반복하지 않고 다른 변형을 제공합니다.

📈 자가 개선 루프

학습자들이 반복해서 막히는 유닛은 자동으로 재생성 대상으로 표시. 콘텐츠가 시간이 지나며 스스로 좋아집니다.

학습 경험

진단 → 학습 → 평가 → 재학습의 닫힌 루프

이미 아는 건 건너뛰고, 모르는 것만, 막히면 다른 방식으로. 망각하기 전에 다시.

1

온보딩

주제·목표·주당 가용시간 입력 → 일정 자동 생성

2

적응형 진단

아는 개념은 스킵, 빈 곳만 학습 경로에

3

학습

P-C-S-M-A 유닛 + 실시간 AI 튜터

4

평가

오답 원인을 공백·오개념·실수로 진단

5

재학습

다른 변형으로 재설명 + 간격 반복

↻ 약점이 사라질 때까지 — 학습자 숙련도(mastery)가 모든 결정의 기준
로드맵

코딩 학습으로 검증하고, 전 분야로 확장

코딩은 '코드 실행으로 정답을 객관 검증'할 수 있어 출발점으로 최적. 학습 루프를 증명한 뒤 어학·교양·자격증으로 넓힙니다.
