React Hooks 완전 정복
핵심 개념 ⏱ 약 40분 진행 중
🤔왜 배워야 할까요?

여러분이 처음 React를 배울 때, 컴포넌트 안에서 데이터를 기억하게 만들려면 어떻게 해야 할지 막막했던 경험이 있으신가요?

예를 들어, 쇼핑몰에서 장바구니 수량 카운터를 만든다고 상상해보세요. "+" 버튼을 클릭할 때마다 숫자가 1씩 올라가야 합니다. 그런데 일반 변수로는 화면이 바뀌지 않습니다. 왜일까요?

또 다른 상황: 사용자가 페이지에 들어왔을 때 서버에서 데이터를 불러오고 싶습니다. 이 작업은 렌더링이 끝난 후 실행돼야 하는데, 어디에 코드를 작성해야 할까요?

바로 이 두 가지 문제를 해결하는 것이 React Hooks입니다. useState와 useEffect만 이해해도 실무 React 개발의 80%를 커버할 수 있습니다.

📖개념 설명

Hook은 함수형 컴포넌트에서 React의 상태(state)와 생명주기(lifecycle) 기능을 사용할 수 있게 해주는 함수입니다. React 16.8부터 도입됐으며, 클래스 컴포넌트 없이도 복잡한 상태 관리가 가능해졌습니다.

① useState — 상태 저장소

const [count, setCount] = useState(0)

컴포넌트 내부에 기억 공간을 만듭니다. count는 현재 값, setCount는 값을 바꾸는 함수입니다. setCount를 호출하면 React가 해당 컴포넌트를 자동으로 다시 렌더링합니다.

② useEffect — 사이드 이펙트 처리

렌더링이 끝난 후 실행할 코드를 등록합니다. API 호출, 타이머, DOM 직접 조작 등 렌더링 외부 세계와 소통하는 모든 작업에 사용합니다.

핵심은 의존성 배열입니다: 배열 안의 값이 바뀔 때만 effect가 재실행됩니다. 빈 배열 []이면 마운트 시 한 번만 실행됩니다.

💻코드 예제

아래 예제를 직접 실행하고 수정해보세요. userId가 바뀔 때만 useEffect가 재실행되는 동작을 확인해보세요.

JavaScript — React
import { useState, useEffect } from 'react'; function UserProfile({ userId }) { // 상태 선언: user 데이터와 로딩 상태 const [user, setUser] = useState(null); const [loading, setLoading] = useState(true); useEffect(() => { // userId가 바뀔 때마다 새 데이터를 가져옴 setLoading(true); fetch(`/api/users/${userId}`) .then(res => res.json()) .then(data => setUser(data)) .finally(() => setLoading(false)); // 클린업: 다음 effect 실행 전 이전 요청 취소 return () => { /* AbortController 등 정리 */ }; }, [userId]); // userId가 의존성 — 바뀌면 effect 재실행 if (loading) return <Spinner />; return ( <div> <h1>{user.name}</h1> <p>{user.email}</p> </div> ); }

🚀왜 중요한가요?

React Hooks는 2019년 React 16.8에서 정식 출시된 이후, 지금은 React 개발의 표준이 됐습니다. 클래스 컴포넌트를 사용하는 코드베이스는 점점 줄어들고 있습니다.

📊 실무 데이터:

• 국내 채용 공고의 95% 이상이 Hooks 이해를 필수 역량으로 요구합니다
• React, Next.js, React Native — 모두 Hooks 기반입니다
• GitHub 상위 React 프로젝트의 98%가 함수형 컴포넌트 + Hooks

💼 커리어 연결:

이 개념을 이해하면 다음 유닛 useContext + Redux가 훨씬 쉬워집니다. 전역 상태 관리의 필요성도 자연스럽게 느끼게 될 거예요. Next.js의 서버 컴포넌트 vs 클라이언트 컴포넌트 구분도 Hooks가 핵심입니다.
