🤔 상황을 상상해보세요.

로그인한 사용자의 정보를 여러 컴포넌트에서 보여줘야 합니다. Header에도, Profile에도, Dashboard에도. 매번 props로 전달하다 보니 코드가 엉망이 됩니다. 이걸 어떻게 해결할 수 있을까요?

Hook은 함수형 컴포넌트에서 React의 상태(state)와 생명주기(lifecycle) 기능을 사용할 수 있게 해주는 함수입니다. 클래스 컴포넌트 없이도 복잡한 상태 관리가 가능해집니다.

가장 중요한 두 Hook은 useState와 useEffect입니다. useState는 컴포넌트 내부 상태를, useEffect는 사이드 이펙트(데이터 요청, 구독 등)를 처리합니다.

import { useState, useEffect } from 'react'; function UserProfile({ userId }) { const [user, setUser] = useState(null); const [loading, setLoading] = useState(true); useEffect(() => { // userId가 바뀔 때만 다시 실행 fetchUser(userId) .then(data => setUser(data)) .finally(() => setLoading(false)); }, [userId]); // 의존성 배열 if (loading) return <Spinner />; return <div>{user.name}</div>; }

React Hooks는 2019년 이후 실무에서 100% 사용되는 핵심 기술입니다. 채용 공고의 95% 이상이 Hooks 이해를 필수 요건으로 요구합니다. 이 개념만 확실히 잡으면 Next.js, React Native도 자연스럽게 연결됩니다.
